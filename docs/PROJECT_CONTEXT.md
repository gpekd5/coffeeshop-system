# Project Context

## 1. 프로젝트명

커피 주문 및 포인트 결제 시스템

---

## 2. 프로젝트 소개

다수의 서버와 인스턴스 환경에서도 안정적으로 동작하는
커피 주문 시스템을 구현한다.

사용자는 메뉴를 조회하고, 포인트를 충전한 뒤,
보유 포인트를 사용하여 커피를 주문할 수 있다.

완료된 주문 정보는 데이터 수집 플랫폼으로 전송하며,
최근 7일간의 주문 데이터를 기준으로 인기 메뉴를 제공한다.

---

## 3. 프로젝트 목표

단순한 CRUD 구현을 넘어 다음 문제를 해결하는 것을 목표로 한다.

- 다중 서버 및 다중 인스턴스 환경 대응
- 포인트 충전 및 사용 과정의 동시성 제어
- 주문과 포인트 데이터의 일관성 보장
- 외부 데이터 수집 플랫폼 전송 실패 대응
- 중복 요청 및 중복 처리 방지
- 기능과 제약사항에 대한 테스트 작성

## 9. 설계 원칙

- 인증된 사용자 식별자는 요청 본문이 아닌 인증 컨텍스트에서 가져온다.
- 금액과 포인트 계산은 서버에서 수행하고 검증한다.
- 포인트 잔액은 음수가 될 수 없다.
- 주문과 포인트 차감은 데이터 일관성이 보장되어야 한다.
- 서버 로컬 메모리에만 의존하는 동시성 제어를 사용하지 않는다.
- 외부 시스템 장애로 인해 완료된 주문 데이터가 유실되지 않도록 한다.
- Entity를 API 요청 또는 응답으로 직접 사용하지 않는다.
- 구현과 문서가 변경되면 함께 갱신한다.
